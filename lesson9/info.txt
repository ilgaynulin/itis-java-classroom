Процедурное программиров - данные и подпрограммы, данные + алгоритмы

ООП - Объектно-ориенторованное программирование
Парадигма программирования, которая представляет программу как совокупность объектов и взаимодействие между ними

Объекта есть состояние и поведение
Состояние:
- модель автомобиля
- объём топливного бака
- Количество бензина
- и т.д.

Поведение
- едет (тратит топливо)
- издавать звуки

Класс - шаблон для создания объекта. Описание будущих объектов
Класс в java - данные (String model, float gasTankVolume) и методы (drive(), honk())
Класс - это ссылочный тип данный

Метод - его поведение


class Vehicle {
	// 
}


Стек (Stack) - Вызванные методы, его аргументы с которыми были вызваны методы, переменные внутри метода, Ссылки на объекты

Куча (Heap) - храняться сами объекты

Конструкторы - это инициализатор объекта. 

У классов неявно есть конструкторы по умолчанию. Все поля класса в нём заполняются значениями по умолчанию.

Можно объявить свой конструктор с нужным количеством аргументов и заполнить ими поля. При этом конструктор по умолчанию отключается

В конструкторе инициализируются поля из аргументов, можно валидировать поля из аргументов, заполнять некоторые поля по умолчанию.
