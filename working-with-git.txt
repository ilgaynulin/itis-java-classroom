Git

Клонирование репозитория
git clone https://github.com/ilgaynulin/itis-java-classroom   // url - ссылка на репозиторий


Удалённый репозиторий - репозиторий на сайте (в данном случае в github)
Локальный репозиторий - то что у нас в компьютере (склонированный репозиторий)

git status - команда для проверки на изменение

добавляем файлы к комиту
git add . // или git add file_name

коммит
git commit -m 'текст описания коммита'

отправка коммита в удалённый репозиторий
git push

получение изменений
git pull

Создание файла .gitignore
В нём содержаться правила игнорирования файлов в системе контроля версии (git)
Пример:
Создаём файл .gitignore в корне репозитория и записываем туда
*.class

Управляющие конструкции
switch(переменная) {
	case 1:
		// выполнение кода если переменная равна 1
		break; // выйти из конструкции switch
	case 2:
		// выполнение кода если переменная равна 2
		break;
	default:
		// если все перечисленные case не подходят
}
